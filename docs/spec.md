## 3. サイトのトーン＆マナー

### 3-1. コピーの方針

- **堅すぎない、読ませる文章** — 不動産サイトにありがちな定型文ではなく、浦松さんの「語り」が感じられるコピー
- **新聞記者出身らしいテンポ** — わかりやすく、リズムのある短文中心
- **「この人に相談したい」と思わせる** — 人柄・経験・実績が自然と伝わる構成
- **多言語対応を前面に** — 外国人対応が最大の武器であることを隠さない

### 3-2. ビジュアルの方向性

- 温かみがありつつ信頼感のあるデザイン
- 「四葉」のモチーフ（幸運・縁起の良さ）を活かす
- 和モダンまたはナチュラル系のトーン（堅いビジネス感は避ける）
- 多言語対応をアイコン・バッジなどで視覚的にも訴求

## 4. ページ構成

ゼットリンカーのサイト構成をベースに、不動産業向けにカスタマイズする。

### 4-1. ページ一覧

| パス | ページ名 | 内容 |
|------|----------|------|
| `/` | トップページ | ヒーロー + サービス概要 + 代表メッセージ + お知らせ + CTA |
| `/about` | 会社概要・代表紹介 | 浦松さんの経歴・想い・会社情報 |
| `/services` | サービス一覧 | 事業内容の概要 |
| `/services/rental` | 賃貸サポート | 外国人向け賃貸物件探しの支援 |
| `/services/sale` | 売買サポート | 物件売買の仲介 |
| `/services/management` | 管理 | 物件管理サービス |
| `/services/foreign-residents` | 外国人向け特設 | 多言語対応・外国人サポートの詳細 |
| `/properties` | 物件一覧 | 取扱物件の検索・一覧 |
| `/properties/[id]` | 物件詳細 | 個別物件の詳細情報 |
| `/column` | コラム一覧 | 不動産お役立ち情報（FB投稿のような実用記事） |
| `/column/[slug]` | コラム詳細 | 個別コラム記事 |
| `/news` | お知らせ一覧 | 会社からのお知らせ |
| `/news/[id]` | お知らせ詳細 | 個別お知らせ |
| `/contact` | お問い合わせ | フォーム（日本語 + 英語対応を想定） |
| `/privacy-policy` | プライバシーポリシー | 個人情報の取り扱い |
| `/terms` | 利用規約 | サイト利用規約 |
| `/legal-notice` | 宅建業法に基づく表記 | 免許番号・所在地等の法定表示 |

### 4-2. 共通レイアウト

- **Header** — ロゴ + ナビゲーション + 言語切替（将来対応） + お問い合わせCTA
- **Footer** — 会社情報 + 免許番号 + リンク集 + SNS
- **管理画面** (`/admin/*`) — Header/Footer非表示（ゼットリンカーと同じ `isAppShell` パターン）

## 6. 技術仕様

### 6-1. ベース構成（ゼットリンカーから継承）

- SEO対応: metadata, OGP, sitemap.ts, robots.ts
- Google Analytics 組み込み
- アクセシビリティ: スキップリンク対応
- Header/Footer の出し分け（isAppShell パターン）
- Admin画面の分離

### 6-2. 追加で必要な機能

| 機能 | 優先度 | 備考 |
|------|--------|------|
| 物件一覧・検索 | 高 | フィルタリング（エリア・間取り・外国人可 等） |
| 物件詳細ページ | 高 | 写真ギャラリー・地図・条件表 |
| お問い合わせフォーム | 高 | Zod バリデーション |
| コラム CMS | 中 | Admin画面から投稿管理 |
| お知らせ CMS | 中 | Admin画面から投稿管理 |
| 多言語対応(i18n) | 低（将来） | まず日本語で構築、後から英語・中国語等を追加 |

### 6-3. データベース（Prisma + Supabase）

主要テーブル（想定）:

- `properties` — 物件情報
- `property_images` — 物件画像
- `columns` — コラム記事
- `news` — お知らせ
- `contacts` — お問い合わせ
- `admin_users` — 管理者
